# Структура

Основные package **restaurant.agents** и **restaurant.utils**

 1. Restaurant.agents содержит классы всех агентов и их behavior'ы. Агенты соответсвуют диаграмме с семинара.
 2. Restaurant.utils содержит класс JsonHandler и SimpleNameFormater, первый является Фасадом для доступа к данным (из json ресурсов) о блюдах, оборудовании, поварах и тд и логирования процессов приготовления каждого блюда в build/resources/main/process_log.json. Стоит отметить, что добавлена обработка и логгирование ошибки при некорректом формате json-данных. SimpleNameFormater отвечает за получения названий для логов.
**Resources** хранят входные json-файлы и конфиг для логера.

# Логирование
Файлы логирования находятся в директории temp/log 

 1. info.log  содержит информацию об операциях, которые производились сущетвующими агентами.
 2. error.log содержит информацию об ошибках, произошедших во время выполнения программы.
# Загрузка json-файлов
В директории utils.json лежат директории с шаблонами классов, которые используются для десериализации объектов, на ряду с аннотациями json,  используемых моделями в пакете restaurant.model. Их структура ааологична формату файлов в задании, за исключенями тех свойств, которые не использовались в программе

# Нововведения
 1. Количество посетителей является константными число и равно 10, они не загружаются из json-файла. Каждый посетитель делает заказ из случайных позиций в меню, а после получения всех блюд из заказа ждёт 10 секунд и совершает новый.
 2. Максимальный параллелизм был достигнут путём взаимодествия агентов исключительно через жёлтые страницы так, что каждый агент ищет первого не занятого агента с нужным типом сервиса и использует его для получения необходимой информации и выполнения действий. Это поведение достигнуто благодаря кастомной реализации агента (SuperAgent).
